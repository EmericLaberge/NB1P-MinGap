# MinGap reproducibility bench

This directory contains the **committed** reproducibility driver for the
MinGap paper's empirical section. The same scripts used to produce the
numbers in the paper live here, alongside the seed derivation helpers and
a frozen baseline of past results.

## Layout

```
bench/
├── README.md         # this file
├── bench_paper.py    # main driver — `python bench/bench_paper.py {part}`
├── bench_paper.json  # results (incremental, resumable; safe to diff against)
└── seeds.py          # canonical seed derivation (seed_for)
```

## Quick start

To reproduce the entire empirical section:

```bash
cd ~/maitrise/redaction/MinGap
uv run python bench/bench_paper.py all
```

This writes results incrementally to `bench/bench_paper.json`. Each part
is **resumable** — re-running the same command skips entries whose key
is already present in the JSON, so an interrupted run can be restarted
without redoing finished work.

To run a single part:

```bash
uv run python bench/bench_paper.py sat       # SAT scaling grid
uv run python bench/bench_paper.py maxsat    # MaxSAT (RC2) scaling
uv run python bench/bench_paper.py fuzz      # cost-agreement validation
uv run python bench/bench_paper.py syntesim  # phylogenetic instances
uv run python bench/bench_paper.py bounds    # LP lower / feasible upper bounds
uv run python bench/bench_paper.py real      # real biological data (OrthoFinder)
```

## Subcommands

| Part      | What it does                                                                 | Trials             |
|-----------|------------------------------------------------------------------------------|--------------------|
| `sat`     | Times CNF build + SAT solve on random and satisfiable ternary matrices       | 500–1000 per cell  |
| `maxsat`  | Times RC2 MinGap solving with per-instance pebble timeouts (parallel pool)    | 5 per cell         |
| `fuzz`    | Brute-force vs MaxSAT cost agreement on small n; correctness smoke test      | 300 per n (n=3..8) |
| `fuzz10`  | Extended fuzz range, n=3..10 (n=10 reduced to 50 trials for wall-time budget) | 300 per n (n=3..9), 50 per n (n=10) |
| `syntesim`| MaxSAT on simulated phylogenetic matrices generated by the syntesim package  | 12 (6 sizes × 2)   |
| `bounds`  | LP-relaxation lower bound, LP-rounding and feasible upper bounds on generated + real data | 17 fixed instances |
| `real`    | Brute-force + MaxSAT on real OrthoFinder outputs (Mycoplasma, 3/4/5 species) | 3 datasets         |
`fuzz` is the most important one for reviewers: it cross-validates
MaxSAT against the brute-force reference, so `mismatches == 0` and
`max_abs_diff == 0` are the canonical invariants.

## Deterministic seeds

All random matrices are generated from a seed computed by
`bench/seeds.seed_for(trial, n, k, gen_offset)`:

```
seed = trial * 7919 + n * 131 + k * 17 + gen_offset
```

The multipliers are primes chosen so adjacent (trial, n, k) values
produce well-separated seeds — no collisions across the parameter grid.
`gen_offset` is 0 for `random` matrices and 1 for `satisfiable` ones
(the two generators in `scripts/benchmark_sat.py`).

The fuzz part calls `seed_for(trial, n)` with `k=0, gen_offset=0`,
matching the legacy inline formula `t * 7919 + n * 131`.

**Do not** duplicate this formula in other scripts — import
`seed_for` from `bench/seeds.py` instead.

## Output format

| Key             | Shape                            | What it holds                                          |
|-----------------|----------------------------------|--------------------------------------------------------|
| `sat_scaling`   | `{key: stats}`                   | `{gen}|n={n}|m={m}` → clause/build/solve means, sat_pct |
| `maxsat_scaling`| `{key: stats}`                   | `{gen}|n={n}|m={m}` → solved_pct, t_mean, t_max, ...   |
| `fuzz`          | `{trials, mismatches, ...}`      | Current fuzz run (n=3..8, 1800 trials)                 |
| `fuzz_n3_n7`    | `{trials, mismatches, ...}`      | Frozen historical baseline (n=3..7, 1500 trials)       |
| `fuzz_n3_n10`   | `{trials, mismatches, ...}`      | Extended fuzz run (n=3..10, n=10 reduced to 50 trials) |
| `syntesim`      | `[row]`                          | Per-instance cost and timing on simulated matrices     |
| `bounds`        | `[row]`                          | Per-instance LP lower bound plus LP-rounding/greedy/MaxSAT/brute-force upper bounds |
| `real`          | `[row]`                          | Per-dataset brute-force + MaxSAT cost/time on real biological matrices |
| `ilp_frontier`  | `[row]`                          | Gurobi ILP on the MaxSAT frontier cells (`bench/ilp_frontier.py`; n ∈ {10, 12}, m = 2n, 5 seeds, 3 s budget) |
The `fuzz_n3_n7` key is preserved as a frozen historical baseline so
reviewers can diff a fresh run against the original. Once the new
extended `fuzz` block is regenerated, it becomes the new canonical
baseline and `fuzz_n3_n7` may be removed in a future cleanup.

## Reproducing the paper

The expected workflow is:

```bash
cd ~/maitrise/redaction/MinGap
uv sync                  # install workspace deps
uv run python bench/bench_paper.py all
```

Compare the resulting `bench/bench_paper.json` against the committed
copy. Every value should match exactly for the fuzz part (deterministic
seeds guarantee this), and within run-to-run noise for the timed parts.

## Implementation notes

- `fuzz_validate` uses `mingap.get("brute_force")` and
  `mingap.get("maxsat")`. Brute force is feasible up to n=8 thanks to
  numba JIT (`mingap._fast.brute_force_njit`); n=9 would need ~362880
  perms × 300 trials and was deliberately not extended.
- `maxsat_scaling` uses `pebble.ProcessPool` for per-instance timeouts.
- `bounds_bench` records `lp_relax_lb`, `lp_round_ub`, `greedy_ub`, `maxsat_ub`, and
  `brute_force_cost` (the latter only for `n <= 10`) for deterministic
  random/satisfiable matrices and every available real dataset.
- `tsp_eggnog_bench` records `tsp_approx_ub` (Christofides/CBM heuristic),
  `greedy_ub`, `lp_round_ub`, and `lp_relax_lb` on the two full-size EggNOG
  datasets (saccharomyces_4890 800×20, bacillus_1386 500×15) under the
  `tsp_approx_eggnog` key; full-size conversions are cached next to the
  shared real-data cache as `<name>_<max_ogs>x<top_n>.csv`.
- `syntesim_instances` writes one row per (n_leaves, n_genes, rep)
  triple; check `nb1p.check(mat)` for the no-backtrack invariant.
- `real_data_bench` runs MinGap on real OrthoFinder outputs from
  Mycoplasma proteomes (3 / 4 / 5 species, hundreds of orthogroups).
  Data is fetched lazily from OrthoFinder's test-suite CSVs and cached at
  `~/.cache/mininv/real/`; if the fetch fails the loader returns an empty
  list and the bench writes an empty `real` key without crashing.
  See `bench/real_data.py` for the source URLs and the count→ternary
  mapping.
